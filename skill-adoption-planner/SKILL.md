# Skill 推广规划器

制定 Skill 在企业内的推广策略，让好工具被更多人用起来。

## 适用场景

| 场景 | 问题 | 输出 |
|------|------|------|
| 新工具引入 | "从哪个部门开始推？" | 部门优先级排序 |
| 推广受阻 | "为什么大家不愿意用？" | 阻力分析 + 应对策略 |
| 规划制定 | "Q2 的推广计划怎么做？" | 分阶段推广计划 |
| 效果评估 | "推广效果怎么样？" | 采用率分析 |

## 工作流程

### 第一步：了解企业情况

询问用户以下信息：

**企业基本信息**
- 企业规模（人数）
- 主要部门构成
- 行业类型

**当前状态**
- 是否已有 AI 工具使用经验？
- 哪些部门对新技术接受度高？
- 是否有 IT/技术支持团队？

**推广目标**
- 希望推广哪些 Skill？
- 目标覆盖人数/部门？
- 期望时间框架？

### 第二步：部门分析

使用 `scripts/analyze_departments.py` 分析各部门特点：

```bash
python scripts/analyze_departments.py --departments "技术,产品,销售,市场,财务,HR"
```

**分析维度**：
1. **技术接受度**：对新技术的开放程度
2. **使用频率潜力**：日常工作中使用 AI 的机会
3. **影响力**：对其他部门的示范效应
4. **推广难度**：需要的培训和支持力度

### 第三步：识别关键角色

**种子用户**（Seed Users）
- 特点：技术敏感、乐于尝试、有影响力
- 作用：早期试用、提供反馈、口碑传播
- 数量：每部门 1-2 人

**关键影响者**（Key Influencers）
- 特点：部门负责人、意见领袖
- 作用：推动采用、资源支持
- 数量：每部门 1 人

**抵触者**（Resistors）
- 特点：保守、担心被替代
- 应对：重点沟通、展示价值

### 第四步：制定推广计划

**推广策略类型**：

| 策略 | 适用场景 | 特点 |
|------|----------|------|
| 技术驱动 | 技术团队强大 | 从技术部门开始，逐步扩展 |
| 业务驱动 | 业务痛点明确 | 从痛点最大的部门开始 |
| 自上而下 | 管理层支持 | 领导带头，全员跟进 |
| 自下而上 | 草根创新 | 员工自发使用，逐步推广 |

**分阶段计划模板**：

```
第一阶段：试点（1-2 周）
- 目标：验证 Skill 可用性
- 范围：1 个部门，5-10 人
- 重点：收集反馈，快速迭代

第二阶段：扩展（2-4 周）
- 目标：扩大使用范围
- 范围：2-3 个部门，20-50 人
- 重点：培训支持，案例积累

第三阶段：推广（4-8 周）
- 目标：全员覆盖
- 范围：所有目标部门
- 重点：持续优化，效果评估
```

### 第五步：预测阻力并准备应对

**常见阻力及应对**：

| 阻力 | 表现 | 应对策略 |
|------|------|----------|
| 学习成本 | "太复杂了，学不会" | 简化入门、视频教程、1对1辅导 |
| 时间压力 | "没时间学新东西" | 展示 ROI、领导支持、渐进式使用 |
| 信任问题 | "AI 不靠谱" | 人工审核、成功案例、渐进式信任 |
| 替代焦虑 | "会不会被 AI 取代" | 强调辅助定位、技能升级机会 |
| 习惯惯性 | "原来的方式挺好" | 对比演示、同事推荐、激励机制 |

## 部门特点参考

### 技术部门

**特点**：
- 技术接受度：🟢 高
- 学习能力：强
- 使用场景：代码生成、文档编写、问题排查

**推广建议**：
- 可作为种子部门
- 强调效率提升
- 鼓励自定义 Skill

### 产品部门

**特点**：
- 技术接受度：🟢 高
- 跨部门协作多
- 使用场景：需求分析、竞品研究、文档撰写

**推广建议**：
- 与技术部门同步推广
- 强调协作效率
- 提供产品相关 Skill

### 销售部门

**特点**：
- 技术接受度：🟡 中
- 结果导向
- 使用场景：客户沟通、方案撰写、数据分析

**推广建议**：
- 用 ROI 数据说话
- 提供即用型 Skill
- 展示成功案例

### 市场部门

**特点**：
- 技术接受度：🟡 中
- 创意需求高
- 使用场景：内容创作、数据分析、活动策划

**推广建议**：
- 强调创意辅助
- 提供内容生成 Skill
- 展示效率对比

### 财务部门

**特点**：
- 技术接受度：🔴 低
- 合规要求高
- 使用场景：报表分析、数据处理、审计辅助

**推广建议**：
- 强调准确性和审核
- 从低风险场景开始
- 领导推动

### HR 部门

**特点**：
- 技术接受度：🔴 低
- 人际沟通多
- 使用场景：简历筛选、培训材料、政策解读

**推广建议**：
- 提供简单易用的 Skill
- 强调辅助而非替代
- 从行政事务开始

## 企业文化适配

### 创新型企业

**特点**：鼓励尝试、容忍失败
**策略**：自下而上、快速迭代
**重点**：激发创新、分享最佳实践

### 稳健型企业

**特点**：流程规范、风险厌恶
**策略**：自上而下、充分验证
**重点**：合规审查、领导背书

### 结果导向型企业

**特点**：KPI 驱动、效率优先
**策略**：业务驱动、ROI 说话
**重点**：量化效果、快速见效

## 示例

### 示例：200 人科技公司推广计划

**输入**：
- 企业：XX 科技公司，200 人
- 部门：技术 50、产品 20、销售 60、市场 30、财务 20、HR 20
- 目标：Q2 完成全员 Skill 基础培训

**输出**：
```
📋 Skill 推广规划
━━━━━━━━━━━━━━━━━━━━━━
企业：XX 科技公司（200人）
目标：Q2 完成全员 Skill 基础培训

🎯 推广策略：技术驱动型

📊 部门优先级：
1. 🟢 技术部（50人）- 种子部门，接受度高
2. 🟢 产品部（20人）- 与技术协作密切
3. 🟡 销售部（60人）- 需展示具体收益
4. 🟡 市场部（30人）- 对创意工具感兴趣
5. 🔴 财务部（20人）- 保守，需领导推动
6. 🔴 HR部（20人）- 需简单易用工具

📅 推广时间线：
- 第1周：技术部内测（10人）
- 第2-3周：技术部全员 + 产品部试用
- 第4-6周：销售部培训 + 案例积累
- 第7-8周：市场部推广
- 第9-10周：财务/HR 部门
- 第11-12周：全员答疑 + 效果评估

⚠️ 预期阻力：
1. 「学不会」→ 视频教程 + 1对1辅导
2. 「没时间」→ ROI 数据 + 领导背书
3. 「不信任」→ 人工审核 + 渐进使用

📈 成功指标：
- 第4周：技术+产品采用率 > 80%
- 第8周：销售+市场采用率 > 60%
- 第12周：全员采用率 > 70%
```

## 注意事项

1. **因地制宜**：每个企业情况不同，计划需要灵活调整
2. **持续沟通**：推广过程中保持与各部门的沟通
3. **快速响应**：遇到问题及时调整策略
4. **庆祝成功**：分享成功案例，激励更多人参与
