# 会议纪要秘书

将会议录音、原始笔记或转录文本转换为结构化的会议纪要，提取关键决议、行动项和待办事项。

## 核心功能

| 功能 | 说明 | 适用场景 |
|------|------|----------|
| 音频转录 | 使用 Whisper 将音频转为文本 | MP3/M4A/WAV 格式录音 |
| 信息提取 | 识别参与者、决议、行动项 | 原始转录文本或笔记 |
| 结构化输出 | 生成标准会议纪要格式 | 所有会议记录场景 |
| 待办清单 | 自动提取任务、负责人、截止日期 | 需要跟进的会议 |

## 使用流程

### 场景一：处理音频录音

```
用户: "帮我整理这个会议录音"
      [提供音频文件路径]

Step 1: 检查音频格式
        支持: MP3, M4A, WAV, FLAC
        不支持: 视频格式需先提取音频

Step 2: 调用转录脚本
        python scripts/transcribe_audio.py <音频文件路径>

Step 3: 读取转录结果
        转录文件位置: <原文件名>_transcript.txt

Step 4: 执行信息提取（见场景二）
```

### 场景二：处理文本记录

```
用户: "整理这份会议记录"
      [提供文本文件或直接粘贴内容]

Step 1: 读取原始内容
        Read <文件路径> 或接收粘贴文本

Step 2: 识别关键信息
        - 会议主题（从开头或标题推断）
        - 参与者（姓名、职位）
        - 时间（如有明确提及）
        - 核心决议（决定、共识、结论）
        - 行动项（任务、负责人、截止日期）
        - 讨论要点（重要观点、争议点）

Step 3: 结构化整理
        使用标准模板（见 references/meeting_templates.md）

Step 4: 生成会议纪要
        输出到: docs/06-会议纪要/Meeting Minutes/
        文件名格式: YYYY-MM-DD-会议主题.md
```

## 信息提取规则

### 参与者识别

| 模式 | 示例 | 提取结果 |
|------|------|----------|
| 直接列举 | "参会人员：Brad强、天生、绛烨" | 提取所有姓名 |
| 发言标记 | "Brad: 我认为..." | 识别为参与者 |
| 职位标注 | "杨福川（华章公司）" | 姓名 + 职位/单位 |

### 决议识别

**关键词触发**：
- 决定、确定、同意、通过
- 统一使用、统一为、改为
- 必须、应该、需要（强制性表述）

**示例**：
```
原文: "决定将书名统一为 Agent Skills"
提取: 核心决议 - 书名统一为 Agent Skills
```

### 行动项识别

**必须包含三要素**：
1. **任务内容**（做什么）
2. **负责人**（谁来做）
3. **截止日期**（什么时候完成）

**识别模式**：
```
模式1: [负责人]: [任务] [时间]
示例: "Brad强: 更新目录到飞书 (本周日前)"

模式2: [任务] - [负责人] - [时间]
示例: "完成初稿 - 各成员 - 下周一"

模式3: 待办清单格式
- [ ] [任务] [@负责人] [截止日期]
```

## 输出格式

使用标准模板（详见 `references/meeting_templates.md`），包含以下部分：

```markdown
# YYYY-MM-DD 会议主题

## 参与者
- [姓名] ([职位/单位])
- ...

## 核心决议

### 1. [决议主题]
[决议内容详细描述]

### 2. [决议主题]
...

## 讨论要点

### [议题1]
- [要点1]
- [要点2]

## 行动项

| 优先级 | 任务内容 | 负责人 | 截止日期 | 备注 |
|--------|----------|--------|----------|------|
| P0 | [任务] | [负责人] | [日期] | [备注] |
| P1 | [任务] | [负责人] | [日期] | [备注] |

## 下次会议
- **时间**: [待定/具体时间]
- **议题**: [待讨论内容]

---

**会议记录整理**: Claude Code
**原始记录**: [原始文件路径]
```

## 优先级判断

| 优先级 | 含义 | 判断依据 |
|--------|------|----------|
| **P0** | 必须立即执行 | 明确标注"紧急"、"立即"、"本周内" |
| **P1** | 重要但不紧急 | 有明确截止日期，但非本周 |
| **P2** | 可选或长期任务 | 无明确截止日期，或标注"待定" |

## 音频转录脚本

**脚本位置**: `scripts/transcribe_audio.py`

**使用方法**:
```bash
# 基础用法
python scripts/transcribe_audio.py meeting_recording.mp3

# 指定输出文件
python scripts/transcribe_audio.py meeting_recording.mp3 -o output.txt

# 指定语言（默认中文）
python scripts/transcribe_audio.py meeting_recording.mp3 --language zh
```

**依赖要求**:
- Python 3.8+
- openai-whisper 包
- ffmpeg（音频格式转换）

**安装命令**:
```bash
pip install openai-whisper
brew install ffmpeg  # macOS
```

## 故障排查

### 问题1: 音频转录失败

**症状**: 运行转录脚本报错

**排查步骤**:
1. 检查音频文件是否存在且可读
   ```bash
   ls -lh <音频文件路径>
   ```

2. 检查 Whisper 是否正确安装
   ```bash
   python -c "import whisper; print(whisper.__version__)"
   ```

3. 检查 ffmpeg 是否安装
   ```bash
   ffmpeg -version
   ```

4. 尝试手动转录测试
   ```bash
   python scripts/transcribe_audio.py <音频文件> --verbose
   ```

### 问题2: 无法识别参与者

**症状**: 生成的纪要中参与者列表为空

**原因**: 原始记录中没有明确的参与者信息

**解决方案**:
1. 手动补充参与者信息
2. 从发言标记中推断（"XXX: 说..."）
3. 询问用户提供参与者名单

### 问题3: 行动项缺少负责人或截止日期

**症状**: 行动项表格中有空白字段

**处理方式**:
1. 标注为 `[待确认]`
2. 在备注中说明："原始记录未明确，需会后补充"
3. 提醒用户补充缺失信息

### 问题4: 会议主题不明确

**症状**: 无法从内容中提取会议主题

**解决方案**:
1. 使用文件名作为主题（如有）
2. 从第一段内容总结主题
3. 询问用户提供会议主题

## 质量检查清单

生成会议纪要后，执行以下检查：

- [ ] **完整性**: 所有关键决议都已记录
- [ ] **准确性**: 决议内容与原始记录一致
- [ ] **可执行性**: 行动项包含任务、负责人、截止日期
- [ ] **格式规范**: 符合标准模板格式
- [ ] **时间标注**: 会议日期、截止日期格式正确（YYYY-MM-DD）
- [ ] **优先级**: P0/P1/P2 标注合理
- [ ] **来源标注**: 原始记录路径已注明

## 注意事项

1. **隐私保护**: 会议内容可能包含敏感信息，确保输出文件权限设置正确
2. **准确性优先**: 不确定的信息标注为 `[待确认]`，不要编造
3. **时间格式**: 统一使用 `YYYY-MM-DD` 格式
4. **文件命名**: 使用 `YYYY-MM-DD-会议主题.md` 格式，避免特殊字符
5. **原始记录**: 始终保留原始记录路径，便于后续核对
