# 消费分类规则

本文档定义了家庭账本管家的消费分类规则和关键词库。

## 分类体系

### 一级分类（8个）

| 类别 | 说明 | 典型场景 |
|------|------|----------|
| 餐饮 | 食品饮料相关支出 | 餐厅、外卖、超市食品、水果零食 |
| 交通 | 出行相关支出 | 打车、公交、地铁、加油、停车 |
| 购物 | 商品购买支出 | 服装、日用品、电子产品、家电 |
| 娱乐 | 休闲娱乐支出 | 电影、游戏、旅游、健身 |
| 教育 | 学习培训支出 | 课程、书籍、培训、考试 |
| 医疗 | 健康医疗支出 | 药品、挂号、体检、保健品 |
| 住房 | 居住相关支出 | 房租、物业、水电、维修 |
| 其他 | 未分类项目 | 无法归入以上类别的支出 |

## 分类算法

### 1. 精确匹配（优先级最高）

直接匹配品牌名或特定场景：
```
"滴滴打车" → 交通
"星巴克" → 餐饮
"京东购物" → 购物
```

### 2. 关键词匹配

统计各类别关键词出现次数，选择得分最高的类别：
```
"今天在超市买了水果和零食" 
→ 餐饮类关键词：超市(1)、水果(1)、零食(1) = 3分
→ 分类：餐饮
```

### 3. 金额推断

结合金额特征辅助判断：
```
"房租 3000元" → 金额大 + "房" → 住房
"打车 15元" → 金额小 + "打车" → 交通
```

### 4. 用户历史学习

记录用户的分类习惯：
```
用户多次将"711"分类为餐饮
→ 系统学习：711 → 餐饮
```

### 5. 默认分类

无法匹配时归入"其他"类别，提示用户手动分类。

## 特殊规则

### 组合消费

一笔消费包含多个类别时，按主要类别分类：
```
"超市购物：牛奶、面包、洗衣液"
→ 主要是食品 → 餐饮
```

### 模糊描述

描述不清晰时，优先选择高频类别：
```
"花了50块" → 无明确类别
→ 根据历史，50元最常见于餐饮/交通
→ 提示用户确认
```

### 跨类别商品

某些商品可能属于多个类别：
```
"买书" → 可能是教育（学习用书）或购物（休闲读物）
→ 默认：教育
→ 允许用户修改
```

## 预算建议

### 标准预算比例（月收入的百分比）

| 类别 | 建议比例 | 说明 |
|------|----------|------|
| 餐饮 | 20-30% | 日常必需，可优化空间大 |
| 交通 | 5-10% | 通勤刚需，相对固定 |
| 购物 | 10-20% | 弹性较大，可控制 |
| 娱乐 | 5-10% | 生活品质，适度即可 |
| 教育 | 5-15% | 投资自己，长期回报 |
| 医疗 | 5-10% | 不可预测，需预留 |
| 住房 | 30-40% | 固定支出，占比最大 |
| 储蓄 | 10-20% | 强制储蓄，财务安全 |

### 超支提醒阈值

- **黄色预警**：达到预算的 80%
- **红色警告**：达到预算的 100%
- **严重超支**：超出预算 20% 以上

## 数据质量

### 必填字段

- 日期（默认今天）
- 金额（必须提取）
- 类别（自动分类或手动指定）

### 可选字段

- 备注（提取或用户补充）
- 支付方式（微信/支付宝/现金/银行卡）
- 商家名称
- 消费地点

### 数据校验

- 金额：必须为正数，最多2位小数
- 日期：不能是未来日期
- 类别：必须在预定义列表中
- 备注：长度不超过100字符
