# Skill ROI 计算器

评估 Skill 的投资回报率，用数据说话。

## 适用场景

| 场景 | 问题 | 输出 |
|------|------|------|
| 向上汇报 | "投入这么多精力做 Skill，值不值？" | ROI 报告 |
| 预算申请 | "开发这个 Skill 需要多少成本？" | 成本估算 |
| 效果评估 | "这个 Skill 用得怎么样？" | 使用统计 |
| 方案对比 | "应该先做哪个 Skill？" | 优先级排序 |

## 工作流程

### 第一步：收集基础信息

询问用户以下信息：

**Skill 基本信息**
- Skill 名称
- 功能描述
- 目标用户群体

**开发成本**（如果是新 Skill）
- 开发人员：谁参与开发？（技术/业务专家）
- 开发时间：预计需要多少小时？
- 人力成本：按小时计算的人力成本

**使用数据**（如果是已有 Skill）
- 使用次数：过去 N 天/月使用了多少次？
- 使用人数：有多少人在用？
- 节省时间：每次使用平均节省多少时间？

### 第二步：计算成本

使用 `scripts/calculate_roi.py` 计算：

```bash
python scripts/calculate_roi.py --mode cost \
  --dev-hours <开发小时数> \
  --hourly-rate <时薪> \
  --experts <专家参与小时数>
```

**成本构成**：
1. **开发成本** = 开发时间 × 时薪
2. **专家成本** = 专家时间 × 专家时薪
3. **维护成本** = 开发成本 × 20%（年度）
4. **总成本** = 开发成本 + 专家成本 + 维护成本

### 第三步：计算收益

```bash
python scripts/calculate_roi.py --mode benefit \
  --usage-count <使用次数> \
  --time-saved <每次节省分钟> \
  --user-count <使用人数> \
  --hourly-rate <用户时薪>
```

**收益构成**：
1. **时间节省** = 使用次数 × 每次节省时间
2. **人力成本节省** = 时间节省 × 平均时薪
3. **质量提升**（可选）= 错误减少带来的成本节省
4. **总收益** = 人力成本节省 + 质量提升

### 第四步：计算 ROI

```bash
python scripts/calculate_roi.py --mode roi \
  --cost <总成本> \
  --benefit <总收益> \
  --period <周期月数>
```

**ROI 公式**：
```
ROI = (总收益 - 总成本) / 总成本 × 100%
回本周期 = 总成本 / 月均收益
```

### 第五步：生成报告

生成完整的 ROI 分析报告，包含：
- 成本明细
- 收益明细
- ROI 计算
- 建议和结论

## ROI 评估标准

| ROI | 评级 | 建议 |
|-----|------|------|
| > 200% | ⭐⭐⭐⭐⭐ 优秀 | 强烈推荐，可推广到其他部门 |
| 100-200% | ⭐⭐⭐⭐ 良好 | 值得投入，建议持续优化 |
| 50-100% | ⭐⭐⭐ 一般 | 可以接受，需要提升使用率 |
| 0-50% | ⭐⭐ 较低 | 需要重新评估，考虑优化或放弃 |
| < 0% | ⭐ 亏损 | 建议停止投入，分析失败原因 |

## 回本周期评估

| 回本周期 | 评级 | 说明 |
|----------|------|------|
| < 1 个月 | 快速回本 | 立即见效，优先投入 |
| 1-3 个月 | 正常 | 合理的投资周期 |
| 3-6 个月 | 较长 | 需要管理层支持 |
| > 6 个月 | 长期投资 | 需要战略层面决策 |

- 预计开发：销售专家 16 小时 + 技术 8 小时

## 注意事项

1. **数据准确性**：ROI 计算依赖准确的使用数据，建议定期统计
2. **隐性收益**：有些收益难以量化（如员工满意度），可作为补充说明
3. **长期价值**：某些 Skill 的价值需要长期观察才能体现
4. **对比基准**：计算节省时间时，要明确对比的是什么（手工操作 vs 使用 Skill）
