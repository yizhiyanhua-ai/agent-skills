# 邮件回复助手

帮助撰写得体、专业的邮件回复，适应不同场景和语气要求。

## 核心功能

| 功能 | 说明 | 适用场景 |
|------|------|----------|
| 场景识别 | 自动识别邮件类型和回复场景 | 所有邮件 |
| 语气分析 | 分析原邮件的正式程度和紧急性 | 所有邮件 |
| 模板匹配 | 选择合适的回复模板 | 常见场景 |
| 个性化生成 | 根据具体情况调整回复内容 | 所有邮件 |
| 得体性检查 | 检查语气、用词是否得体 | 所有邮件 |

## 使用流程

```
用户: "帮我回复这封邮件"
      [提供原始邮件内容]

Step 1: 分析原始邮件
        - 识别发件人身份（客户/同事/上级/合作伙伴）
        - 判断邮件类型（询问/投诉/邀请/感谢/通知）
        - 分析语气（正式/非正式、紧急/常规）

Step 2: 确定回复场景
        - 商务洽谈
        - 客户投诉
        - 会议邀请
        - 拒绝请求
        - 感谢致谢
        - 日常沟通

Step 3: 选择回复模板
        从 references/email_templates.md 选择合适模板

Step 4: 生成个性化回复
        - 填充具体信息
        - 调整语气和用词
        - 添加必要的细节

Step 5: 得体性检查
        - 语气是否匹配
        - 用词是否恰当
        - 格式是否规范
```

## 场景识别规则

### 商务洽谈

**关键词**：合作、洽谈、方案、报价、商讨

**回复要点**：
- 表达合作意愿
- 提供具体信息或建议时间
- 保持专业和积极的态度

### 客户投诉

**关键词**：问题、不满意、投诉、失望、退款

**回复要点**：
- 首先表达歉意和理解
- 说明问题原因（如已知）
- 提供解决方案
- 承诺改进措施

### 会议邀请

**关键词**：会议、邀请、参加、时间、议程

**回复要点**：
- 明确表示接受或婉拒
- 如接受，确认时间和地点
- 如婉拒，说明原因并建议替代方案

### 拒绝请求

**关键词**：请求、帮忙、能否、可以吗

**回复要点**：
- 感谢对方的信任
- 委婉说明无法满足的原因
- 提供替代建议（如可能）
- 保持友好和尊重

### 感谢致谢

**关键词**：感谢、谢谢、帮助、支持

**回复要点**：
- 表达不客气
- 强调合作关系
- 表示愿意继续提供帮助

## 语气分析

### 正式程度判断

| 指标 | 正式 | 非正式 |
|------|------|--------|
| 称呼 | 尊敬的XX先生/女士 | XX你好 |
| 用词 | 贵司、敝司、拜访 | 你们公司、我们、见面 |
| 结尾 | 此致敬礼、顺颂商祺 | 祝好、期待回复 |

### 紧急程度判断

| 指标 | 紧急 | 常规 |
|------|------|------|
| 标题 | 【紧急】、【重要】 | 无特殊标记 |
| 用词 | 尽快、立即、紧急 | 方便时、有空时 |
| 截止时间 | 今天、明天 | 本周、下周 |

## 回复模板示例

### 模板1：商务合作回复

```
[称呼]您好，

感谢您对[我司/我们]的关注和信任。

关于您提到的[合作事项]，我们非常感兴趣。[具体回应内容]。

为了更好地推进合作，建议我们安排一次详细沟通。您看[时间建议]是否方便？

期待您的回复。

此致
敬礼

[署名]
[日期]
```

### 模板2：客户投诉回复

```
[称呼]您好，

非常抱歉给您带来了不便，我们对此深表歉意。

关于您反馈的[问题]，我们已经进行了调查。[问题原因说明]。

为了解决这个问题，我们将采取以下措施：
1. [措施1]
2. [措施2]

同时，我们会[改进承诺]，避免类似问题再次发生。

再次为给您带来的不便表示歉意，感谢您的理解和支持。

祝好

[署名]
[日期]
```

### 模板3：婉拒请求

```
[称呼]您好，

感谢您的信任和邀请。

关于您提到的[请求事项]，我们经过慎重考虑，[婉拒原因]，因此暂时无法满足您的要求。

不过，我们建议您可以[替代方案]，这可能更适合您的需求。

希望您能理解，也期待未来有机会合作。

祝好

[署名]
[日期]
```

## 文化适配

### 中文邮件特点

- 开头常用"您好"
- 结尾常用"此致敬礼"、"祝好"
- 较为含蓄，避免直接拒绝
- 重视关系维护

### 英文邮件特点

- 开头常用"Dear" 或 "Hi"
- 结尾常用"Best regards"、"Sincerely"
- 较为直接，但保持礼貌
- 重视效率和清晰度

## 质量检查清单

生成邮件回复后，执行以下检查：

- [ ] **称呼正确**: 使用恰当的称呼方式
- [ ] **语气匹配**: 与原邮件语气相符
- [ ] **内容完整**: 回应了所有关键问题
- [ ] **用词得体**: 无不当或冒犯性用词
- [ ] **格式规范**: 段落清晰、结构完整
- [ ] **署名完整**: 包含姓名、职位、日期

## 注意事项

1. **隐私保护**: 不要在邮件中泄露敏感信息
2. **准确性**: 确保事实信息准确无误
3. **及时性**: 根据紧急程度及时回复
4. **专业性**: 保持专业和礼貌的态度
5. **个性化**: 避免过于模板化，增加个性化内容
